What is the ASCII code and why is it important?
The ASCII code is a Latin-based character encoding standard created in 1963 by the American Committee on Standards. It maps 256 integers (representable in a byte) to characters. For example:
  • "a" has a code of 97 and "A" has a code of 65.
  • The digit "0" has a code of 48, and digits continue up to "9" with 57.
ASCII organizes lowercase and uppercase letters sequentially:
  • Lowercase: "a" = 97, "b" = 98, ... "z" = 122.
  • Uppercase: "A" = 65, "B" = 66, ... "Z" = 90.

On the keyboard, you can use ALT + Num to insert a character from the ASCII table (e.g., ALT + 169 = ©).

In Python:

  • print(chr(65)) → 'A'
  • print(ord("A")) → 65
What are the main differences between while and for loops in Python?
  • Condition Type: A while loop is based on a condition, while a for loop iterates over a sequence or range.
  • Number of Repetitions: A while loop depends on a condition and may repeat 0 or more times, while a for loop typically iterates a known number of times.
  • Counter: In a while loop, the counter must be explicitly initialized and updated. In a for loop, the counter is managed by the loop itself.
  • Equivalence: A for loop can always be rewritten as a while loop, but not all while loops can be rewritten as for loops.

How do you use a method in Python?
To use a method in Python, you call it on an object of the class. For example, in the Coordinate class: 
class Coordinate:
    def __init__(self, x, y):
        self.c_x = x
        self.c_y = y
    def distance(self, other):
        x_diff_sq = (self.c_x - other.c_x)**2
        y_diff_sq = (self.c_y - other.c_y)**2
        return (x_diff_sq + y_diff_sq)**0.5
        
# Conventional use:
c = Coordinate(3, 4)
origin = Coordinate(0, 0)
print(c.distance(origin))
# Method called on object 'c'
        
# Equivalent use:
print(Coordinate.distance(c, origin))  
# Method called using class name
In the conventional use, self is automatically passed. In the equivalent use, you explicitly pass the object as the first argument.

How do you create and use a class to represent a number as a fraction in Python?
To represent a number as a fraction, you can create a class called Fraction with the following attributes and methods:
  • Attributes: numerator and denominator (internal representation)
  • Methods:
    • __add__ and __sub__: For adding and subtracting fractions
    • __str__: Converts the fraction to a string (e.g., "3/4")
    • frac2float: Converts the fraction to a float
    • inv: Returns the inverse of the fraction
Example: 
class Fraction:
    def __init__(self, x, y):
        self.num = x
        self.den = y 
    def __add__(self, other):
        num = (self.num * other.den + 
        other.num * self.den)
        den = (self.den * other.den)
        return Fraction(num, den)
    def __sub__(self, other):
        num = (self.num * other.den - 
        other.num * self.den)
        den = (self.den * other.den)
        return Fraction(num, den) 
    def __str__(self):
        return str(self.num) + "/"
        + str(self.den)  
    def frac2float(self):
        return self.num / self.den
    def inv(self):
        return Fraction(self.den, self.num)

Getter and Setter methods are used to access and update private data attributes of a class, maintaining encapsulation.
  • Getter: A method that retrieves the value of a private attribute.

What is a class hierarchy in object-oriented programming?
A class hierarchy is an organizational structure where classes have a parent-child relationship:
  • Parent Class (Superclass): The class from which attributes and methods are inherited.
  • Child Class (Subclass): The class that inherits from the parent class.
Subclasses can:
  • Add new attributes and methods.
  • Redefine (override) methods inherited from the parent class.
Hierarchies allow for code reuse and the extension of class functionality.
How do you create a subclass in Python?
To create a subclass in Python:
  1. Use the parent class from which to inherit attributes and methods.
  2. Define the __init__ method of the subclass, calling the parent's __init__ to initialize inherited attributes and adding new ones.
  3. Add new attributes and methods specific to the subclass.
  4. Override existing methods to customize behavior (polymorphism).
Example: 
class Developer(Employee):
    def __init__(self, name, surname, 
    prog_language):
        Employee.__init__(self, name, surname)
        self.prog_language = prog_language
    def get_prog_language(self):
        return self.prog_language
    def set_prog_language(self, prog_language):
        self.prog_language = prog_language
    def __str__(self):
        return "Dev: " + self.surname + 
        "\\n Lang: " + self.prog_language
Here, Developer inherits from Employee and adds new functionality.
How can different types of bank accounts be modeled using classes in Python?
To model different types of bank accounts, you can create a class hierarchy:
  • CurrentAccount: Basic account with no return.
  • RemuneratedAccount: Inherits from CurrentAccount, with a small monthly return using compound interest.
  • FixedAccount: Inherits from RemuneratedAccount, with a higher return but restrictions on withdrawals. Uses simple interest.
Example Code: 
class CurrentAccount:
  def __init__(self, name, capital=0):
      self.name = name
      self.capital = capital
  def compute_return(self, months):
      return 0

class RemuneratedAccount(CurrentAccount):
  def __init__(self, name, capital, interest):
      super().__init__(name, capital)
      self.interest = interest
  def compute_return(self, months):
      return self.capital * 
    ((1 + (self.interest / 100))**months - 1)

class FixedAccount(RemuneratedAccount):
  def __init__(self, name, capital, 
  interest, period):
      super().__init__(name, capital, interest)
      self.period = period
  def compute_return(self, months):
      return self.capital * 
      (self.interest / 100) * months
